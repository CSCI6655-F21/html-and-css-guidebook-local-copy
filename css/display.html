The Display Property

The display property determines the basic layout behavior of an element. Most elements have a default value of either block or inline.

Block

Block behavior makes elements stack one-below-another. Think of it like upside-down gravity: the elements fall to the top of the page and land below the one that came before. They also stretch horizontally to fill all the space available in the container.

Note that in this example, space has been added between the divs to make them easier to see.

div {
	background-color: lightblue;
	height: 2rem;
	margin-bottom: .2rem;
}
<div></div>
<div></div>
<div></div>

Headings, paragraphs, divs, and many more elements behave as block by default.

<h1>Heading 1</h1>

<p>A paragraph of text.</p>

<h2>Heading 2</h2>

<div>Some text in a div.</div>

Note that even though none of the elements in the above example contain enough text to fill all the horizontal space available, each is surrounded by an invisible box that extends all the way to the right edge of the container. We can make that box visible by adding a background color.

Note that in this exmaple the space between the elements results from the browser's default margin for those elements.

h1, h2, p, div {
	background-color: lightblue;
}

Inline-Block

We run into issues when we try to apply vertical (top & bottom) padding, margin, or border to inline elements. In order to preserve the position of the text on a line, those properties don't affect the element's position:

strong {
	background-color: lightcoral;
	border: 2px solid lightgreen;
	margin: 1rem;
	padding: 1rem;
}
<p>This is a long paragraph of example text that wraps to multiple lines, and the <strong>strong</strong> tag has padding, margin, and border applied. Notice how they work as expected on the horizontal axis, but don't cause the element to reposition on the vertical axis, which in turn causes weird overlapping.</p>

We could use display: block;, but that would cause the element to stretch horizontally all the way across the container:

strong {
	background-color: lightcoral;
	border: 2px solid lightgreen;
	display: block;
	margin: 1rem;
	padding: 1rem;
}
<p>This is a long paragraph of example text that wraps to multiple lines, and the <strong>strong</strong> tag has padding, margin, and border applied, and is set to display: block. Notice how it now stretches horizonally across the container, pushing the neighboring content to the top and bottom. </p>

So instead we can use the hybrid display: inline-block;, which keeps the limited width and sits in a horizontal line like text, but respects vertical padding, margin, and border:

strong {
	background-color: lightcoral;
	border: 2px solid lightgreen;
	display: inline-block;
	margin: 1rem;
	padding: 1rem;
}
<p>This is a long paragraph of example text that wraps to multiple lines, and the <strong>strong</strong> tag has padding, margin, and border applied, and is set to display: inline-block. Notice how it keeps its basic width and positioning but now respects vertical padding, margin, and border.. </p>

The above example is unlikely to be a common scenario for use of inline-block, but it can come in handy for all sorts of reasons, such as when styling links to look like buttons:

a {
	background-color: lightcoral;
	border-radius: .5rem;
	box-shadow: 2px 2px #444;
	display: inline-block;
	margin: .5rem;
	padding: 1rem 2rem;
}

Display: None

Occasionally, we want to visually remove an element from the page entirely, inlcuding any effect it might have on the rest of the page's layout. Examples might include a modal dialog that is hidden until a user takes an action to show it, or the contents of a collapsed accordion element, on the contents of a dropdown menu when not in use. For this, we can use display: none;, which causes the element to behave as though it doesn't exist at all.

p {
	display: none;
}

Other Display Values

There are a number of other possible values for the display property, including display: flex; and display: grid;, but those serve an entirely different purpose and will be found in the articles about Flexbox and CSS Grid repectively.
